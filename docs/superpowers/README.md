# Superpowers

Design specifications and implementation plans for contradiction-checker features.

## Active

In-progress work that has not yet shipped. These are the source of truth for
ongoing development.

- `plans/2026-06-02-definition-detector-eval.md` — Definition-detector eval (judge P/R/F1 + production precision)
- `specs/2026-06-02-definition-detector-eval-design.md` — Design spec for the above

## Archive

Shipped work, kept for provenance. Each file in `archive/` maps to a completed
ADR (see `../../decisions/`) and a CHANGELOG entry. Use these to understand
the step-by-step build that produced a shipped feature.

- `archive/plans/` — implementation plans for shipped features
- `archive/specs/` — design specs for shipped features

## Convention

Each plan/spec follows the `YYYY-MM-DD-<feature>.md` naming convention. The
date is the authoring date, not the ship date. Ship date lives in the
CHANGELOG.
