# v0.2 Build Plan

## Context

v0.1.0 shipped a working two-stage pipeline against `.txt`/`.md` corpora. The roadmap in [`futureplans.md`](../../futureplans.md) lists the obvious gaps; this plan turns three of them into a PR-sized build sequence:

1. **PDF + DOCX loaders** — replace the `NotImplementedError` stubs in `consistency_checker/corpus/loader.py`.
2. **Numeric / quantitative extractor** — short-circuit the LLM judge on sign-flip contradictions, side-channel the rest to the judge prompt.
3. **Three-document conditional contradictions** — find `A ∧ B ⇒ ¬C` triangles that pairwise scans can't catch.

13 steps across three blocks. Estimated ~10.5 engineer-days, ~8 calendar days if D / E / F interleave.

## Decisions (confirmed)

1. **Loader library**: `unstructured` for both PDF and DOCX (and any other formats it handles). Single dependency, single registry entry, one set of char-span bookkeeping to write and test. Trade-off: heavy install footprint (pulls in pdfminer, magic, optional OCR). Pin minimal extras (`unstructured[local-inference]` or narrower) to keep wheels tractable.
2. **Numeric design**: pre-judge short-circuit (A). Sign-flip pairs resolved deterministically; LLM judge never called. New verdict label `numeric_short_circuit` introduced into `JudgeVerdictLabel`.
3. **Three-doc approach**: graph-triangle on FAISS-similarity edges. Reuses gate output; scales as O(edges · avg-degree); no entity NER (deferred to v0.3). Known limitation: misses triangles whose edges fall below the gate threshold; document and revisit after CONTRADOC sweep.
4. **Multi-party finding shape**: new `multi_party_findings` table rather than overloading `findings` with a nullable `parent_finding_id`. The pair-shaped report stays untouched; multi-party gets its own renderer.
5. **Build order**: all three blocks, suggested merge order: `D0 → D1 → D2 → D3 → E0 → E1 → E2 → E3 → F0 → F1 → F2 → F3 → F4`. ~9.5 engineer-days total (D shortened by collapsing PDF+DOCX into one loader).

## Block D — PDF and DOCX loaders (via `unstructured`)

**Effort: 2–3 engineer-days. Critical path: D2 (`unstructured` loader with char-span invariant).**

### D0 — ADR-0004: `unstructured` as the loader backend
- New `docs/decisions/0004-pdf-docx-loaders.md` recording: single backend (`unstructured`) for PDF + DOCX (and other formats it supports for free); body-content elements only (`NarrativeText`, `Title`, `ListItem`, `Table`); headers/footers/footnotes deferred; element-level offset map stored in `documents.metadata_json` under `"element_spans"`; install-extras pinned to the smallest viable set.
- Files: `docs/decisions/0004-pdf-docx-loaders.md` (new).
- Verify: ADR lints; no code, no tests.
- Depends: decision 1 confirmed.

### D1 — Loader-extension protocol + registry refactor
- Extract dispatch in `corpus/loader.py` into `LOADERS: dict[str, FileLoader]`. `.txt`/`.md` becomes `PlaintextLoader`. Also adds `AuditLogger.most_recent_run()` and rewires `cli/main.py:140` to use it.
- Files: `consistency_checker/corpus/loader.py`, `consistency_checker/audit/logger.py`, `consistency_checker/cli/main.py`, `tests/test_corpus.py`, `tests/test_audit_logger.py`, `tests/test_cli.py`.
- Verify: hermetic registry tests; char-span invariant still holds for plaintext; CLI report no longer accesses `store._conn`.
- Depends: D0.

### D2 — `UnstructuredLoader` for PDF and DOCX
- Single loader plugging into D1's registry. Calls `unstructured.partition.auto.partition(file_path)` to get a list of `Element` objects. Concatenate `element.text` for body-content element types in document order, separated by `"\n\n"`. While concatenating, build sidecar `[(element_index, element_type, char_start, char_end), …]` stored as JSON in `documents.metadata_json["element_spans"]`. Table elements are flattened row-major with `\t` / `\n` separators (matching what we'd have done in DOCX-only land) so reports stay diffable. Char-span invariant survives because the text the chunker sees is exactly what was concatenated.
- Files: `consistency_checker/corpus/loader.py`, `pyproject.toml` (add `unstructured` with the minimal extras for PDF + DOCX — pin precisely after a `uv lock` resolution), `tests/fixtures/sample_docs/short.pdf` (new, ≤ 2 pages), `tests/fixtures/sample_docs/short.docx` (new), `tests/test_corpus.py`.
- Verify: hermetic test asserting `text[chunk.char_start:chunk.char_end] == chunk.text` for every chunk from the PDF fixture **and** the DOCX fixture. Hermetic test that element-spans cover `[0, len(text))` with no gaps. Deterministic 2×2 table flatten test from the DOCX fixture so the report golden stays stable. If `unstructured` install footprint forces a `slow` mark (e.g. on first model download for layout inference), mark the import-time tests accordingly — prefer the offline / fast-strategy code path so this stays in default CI.
- Depends: D1.

### D3 — End-to-end loader test + docs
- Add the PDF and DOCX fixtures to `tests/fixtures/e2e_corpus/`; assert fixture-mode pipeline runs unchanged on the mixed corpus. Update README, ARCHITECTURE, CHANGELOG.
- Files: `tests/fixtures/e2e_corpus/06_*.pdf`, `tests/fixtures/e2e_corpus/07_*.docx`, `tests/test_e2e.py`, `README.md`, `docs/ARCHITECTURE.md`, `CHANGELOG.md`.
- Verify: e2e fixture passes hermetically; report golden-file stable on the mixed-format corpus.
- Depends: D2.

## Block E — Numeric / quantitative extractor

**Effort: 2–3 engineer-days. Critical path: E2 (short-circuit wiring + verdict literal expansion).**

### E0 — ADR-0005: numeric short-circuit (design A)
- New `docs/decisions/0005-numeric-short-circuit.md` recording design A, the `numeric_short_circuit` verdict, and the non-goal "no LLM metric canonicalisation in v0.2."
- Depends: decision 3 confirmed.

### E1 — `extract/quantitative.py` extractor module
- New `consistency_checker/extract/quantitative.py` exposing `QuantitativeTuple(metric, value, unit, polarity, scope)` and `extract_quantities(text) -> list[QuantitativeTuple]`. Regex + verb-direction lexicon (`grew/rose/increased → up`; `fell/declined → down`). Unit normalisation via `pint`. Scope is a span-heuristic noun-phrase head (no NER — that's v0.3). Pure function.
- Files: `consistency_checker/extract/quantitative.py` (new), `pyproject.toml` (`pint>=0.23`), `tests/test_quantitative.py` (new).
- Verify: ≥ 15 table-driven cases including sign-flip pairs, range-overlap pairs, unit-mismatch pairs, empty-prose case.
- Depends: E0.

### E2 — Wire short-circuit into the pipeline + extend verdict label
- In `pipeline.check`, after NLI passes threshold but before judge call, run `extract_quantities` on both assertions. If both yield same `metric+scope+unit` with opposite sign → emit `JudgeVerdict(verdict="numeric_short_circuit", confidence=1.0, rationale=…)` and skip the LLM. Extend `JudgeVerdictLabel` Literal in `providers/base.py`. Report renderer treats short-circuit as contradiction in the summary.
- Files: `consistency_checker/pipeline.py`, `consistency_checker/check/providers/base.py`, `consistency_checker/check/llm_judge.py` (type only), `consistency_checker/audit/report.py`, `tests/test_llm_judge.py`, `tests/test_report.py`, `tests/test_e2e.py`.
- Verify: hermetic test with a fixture corpus containing one sign-flip pair; assert `FixtureJudge` is not called for that pair. Golden-file report extended.
- Depends: E1. Touches same `pipeline.check` block as F4 — sequence E2 before F4.

### E3 — Range-overlap "uncertain" hint passed to judge prompt
- When E2 finds same-metric tuples that don't sign-flip but disagree by more than `Config.numeric_disagreement_threshold` (default 0.10), pass a structured `numeric_context` JSON block into the judge user prompt. Existing pure-prose pairs render an empty placeholder so golden prompts stay stable.
- Files: `consistency_checker/pipeline.py`, `consistency_checker/check/llm_judge.py` (`render_user_prompt` gains optional `numeric_context`), `consistency_checker/check/prompts/judge_user.txt`, `consistency_checker/config.py`, `tests/test_llm_judge.py`, `tests/test_config.py`, `CHANGELOG.md`.
- Verify: hermetic test that the rendered prompt contains the numeric block iff there's a numeric mismatch above threshold.
- Depends: E2.

## Block F — Three-document conditional contradictions

**Effort: 4–5.5 engineer-days. Critical path: F3 (triangle judge + prompt design). Biggest risk: F3 prompt quality on real corpora.**

### F0 — ADR-0006: three-doc graph-triangle + `multi_party_findings` table
- New `docs/decisions/0006-three-doc-conditional.md` recording: graph-triangle on FAISS edges; opt-in via `--deep`; new `multi_party_findings` table rather than overloading `findings`; new verdict label `multi_party_contradiction`; entity-NER deferred to v0.3.
- Depends: decisions 4, 5 confirmed.

### F1 — Migration 0003: `multi_party_findings` table
- `consistency_checker/index/migrations/0003_multi_party.sql` with `finding_id PK`, `run_id FK`, `assertion_ids_json` (≥ 3), `doc_ids_json` (≥ 2 distinct), `triangle_edge_scores_json`, `judge_verdict`, `judge_confidence`, `judge_rationale`, `evidence_spans_json`, `created_at`. Indexes on `run_id` and `judge_verdict`. Extend `AuditLogger` with `record_multi_party_finding` / `iter_multi_party_findings`. **Never edit existing migrations.**
- Files: migration file (new), `consistency_checker/audit/logger.py`, `tests/test_audit_logger.py`.
- Verify: round-trip insert/read; JSON columns deserialise; migration is idempotent.
- Depends: F0.

### F2 — Triangle detector on gate edges
- New `consistency_checker/check/triangle.py` exposing `find_triangles(pairs, *, max_per_run=1000) -> Iterator[Triangle]`. Build adjacency from gate output; enumerate triangles; dedupe by sorted assertion-id tuple; cap per-run by min edge score desc. Require at least 2 distinct documents per triangle.
- Files: `consistency_checker/check/triangle.py` (new), `tests/test_triangle.py` (new).
- Verify: clique-of-4 yields 4 triangles; chain-of-3 yields 0; same-doc-only triangles excluded; deterministic given fixed adjacency.
- Depends: F1.

### F3 — Multi-party judge call + prompt
- New `consistency_checker/check/prompts/judge_multi.txt` asking "given A, B, C, does any subset jointly contradict another?" New `MultiPartyJudgePayload` Pydantic model with `verdict ∈ {multi_party_contradiction, not_contradiction, uncertain}`, `confidence`, `rationale`, `evidence_spans`, `contradicting_subset: list[str]`. Sibling `MultiPartyJudge` Protocol so the pairwise type stays narrow. `FixtureMultiPartyJudge` for hermetic tests. `AnthropicMultiPartyProvider` / `OpenAIMultiPartyProvider`. Recommend a new `consistency_checker/check/multi_party_judge.py` module so `llm_judge.py` stays focused.
- Files: prompt file (new), `consistency_checker/check/providers/{base,anthropic,openai}.py`, `consistency_checker/check/multi_party_judge.py` (new), `tests/test_multi_party_judge.py` (new).
- Verify: hermetic happy-path + validation-failure-retry against `FixtureMultiPartyJudge`. `live`-marked Anthropic test against a hand-built A/B/C triangle.
- Depends: F1, F2.

### F4 — Pipeline integration + `--deep` CLI flag + report
- Extend `pipeline.check` with `multi_party_judge: MultiPartyJudge | None`. When present, after the pairwise pass, call `find_triangles` on the same gate output and judge each triangle. Add `Config.enable_multi_party: bool = False`. Add `--deep` flag to `consistency-check check`. Extend report renderer with a "## Multi-document conditional contradictions" section. Update `CheckResult` with `n_triangles_judged` and `n_multi_party_findings`. Add `make_multi_party_judge` factory.
- Files: `consistency_checker/pipeline.py`, `consistency_checker/cli/main.py`, `consistency_checker/config.py`, `consistency_checker/audit/report.py`, `tests/test_e2e.py`, `tests/test_report.py`, `tests/test_cli.py`, new triangle fixture corpus, `CHANGELOG.md`, `README.md`, `docs/ARCHITECTURE.md`.
- Verify: 3-doc fixture where A "all employees get 4 weeks vacation", B "engineers are employees", C "engineers get 2 weeks vacation" — with `--deep`, expect one multi-party finding; without, expect zero. Golden report stable.
- Depends: F3 and E2 (both touch `pipeline.check`).
